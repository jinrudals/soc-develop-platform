# SoC PM 팀

## 개요

SoC PM(Project Management) 팀은 SoC 프로젝트 전반의 일정, 인원, 구조를 관리하고, 각 팀의 진행 현황을 취합하여 프로젝트가 목표대로 진행될 수 있도록 조율하는 업무를 담당한다.

---

## 관리 대상

### 프로젝트 구성

| 항목 | 설명 |
|---|---|
| 프로젝트 목록 | 어떤 프로젝트가 존재하는가 |
| 참여 인원 | 해당 프로젝트에 참여하는 인원은 누구인가 |
| 인원별 역할 | Manager / Project Leader / Engineer 등 (프로젝트마다 다를 수 있음) |
| Milestone | 프로젝트에 어떤 Milestone이 있으며, 각 Milestone의 일정은 어떻게 되는가 |

### Repository 및 RTL Module 구성

| 항목 | 설명 |
|---|---|
| Repository 목록 | 어떤 Repository가 있는가 (하나의 Repository가 여러 프로젝트에서 사용될 수 있음) |
| RTL Module 목록 | 각 Repository에 어떤 RTL Module이 있는가 |
| RTL Hierarchy | RTL Module 간의 연결 관계도 |
| RTL Module 담당자 | 한 RTL Module에 여러 담당자가 있을 수 있으며, 한 사람이 여러 RTL Module을 담당할 수 있음 |

### 프로젝트 현황

- 각 팀(RTL, Verification, DFT, SDC, PI, PD)의 진행 현황은 어떻게 되는가
- Milestone 기준으로 각 팀의 목표 대비 실제 진행도는 어떻게 되는가
- 현재 이슈 및 리스크는 무엇인가
